# 项目技术架构文档

## 架构概述

本项目是一个集成了语音交互功能的电子商城系统，采用前后端分离架构，包含三个主要部分：

1. **React Native前端**：移动应用前端，提供用户界面和交互
2. **Express主后端**：处理核心业务逻辑，如用户管理、商品、订单等
3. **语音服务后端**：处理语音识别、语音合成和AI对话功能

## 系统架构图

```
+-----------------+     +------------------+     +------------------+
|                 |     |                  |     |                  |
|  React Native   |<--->|  Express主后端   |<--->|    MongoDB数据库  |
|   移动应用前端   |     |  (hello_after)   |     |                  |
|    (Hello)      |     |                  |     |                  |
|                 |     +------------------+     +------------------+
|                 |            ^
|                 |            |
|                 |     +------v-----------+     +------------------+
|                 |     |                  |     |                  |
|                 |<--->|  语音服务后端     |<--->|  第三方AI/语音服务 |
|                 |     |    (gzl)         |     |  (讯飞/Coze)     |
|                 |     |                  |     |                  |
+-----------------+     +------------------+     +------------------+
```

## 前端架构（Hello）

前端采用React Native框架开发，使用以下技术和架构模式：

### 技术栈

- **React Native**：跨平台移动应用框架
- **React Navigation**：导航管理
- **Axios**：HTTP请求处理
- **Expo音频模块**：音频录制和播放

### 目录结构

```
/Hello
├── src/
│   ├── components/         # 可复用组件
│   │   └── VoiceAssistant.js  # 语音助手组件
│   ├── screens/            # 应用屏幕
│   │   ├── Home/           # 首页相关屏幕
│   │   ├── Cart/           # 购物车相关屏幕
│   │   ├── Profile/        # 用户相关屏幕
│   │   ├── Message/        # 消息相关屏幕
│   │   └── Video/          # 视频相关屏幕
│   ├── navigation/         # 导航配置
│   ├── services/           # API服务
│   │   ├── api.js          # API配置和拦截器
│   │   ├── user.service.js # 用户相关服务
│   │   ├── product.service.js  # 商品相关服务
│   │   └── order.service.js    # 订单相关服务
│   ├── assets/             # 静态资源
│   └── utils/              # 工具函数
└── App.tsx                 # 应用入口
```

### 数据流

前端采用单向数据流模式：

1. 用户交互触发服务调用
2. 服务通过API与后端通信
3. 组件接收响应并更新状态
4. 界面根据新状态重新渲染

## 后端主服务架构（hello_after）

后端主服务负责处理核心业务逻辑，使用Express框架和MongoDB数据库。

### 技术栈

- **Node.js**：JavaScript运行时
- **Express**：Web应用框架
- **Mongoose**：MongoDB对象建模
- **JWT**：用户身份验证

### 目录结构

```
/hello_after
├── app.js                # 应用入口
├── bin/
│   └── www               # 服务启动脚本
├── routes/
│   ├── index.js          # 主路由
│   └── users.js          # 用户路由
├── database/
│   ├── db.js             # 数据库连接
│   ├── user.js           # 用户模型
│   ├── Merchant.js       # 商家模型
│   └── Logistics.js      # 物流模型
└── public/               # 静态资源
```

### 数据模型关系

```
+-------------+      +--------------+      +------------+
|   User      |<---->|   Order      |<---->|  Product   |
+-------------+      +--------------+      +------------+
      ^                     |                   ^
      |                     |                   |
      |                     v                   |
+-------------+      +--------------+      +------------+
|   Cart      |      |  Logistics   |<---->|  Merchant  |
+-------------+      +--------------+      +------------+
```

1. **User**：用户模型，包含账户、个人信息和地址等
2. **Product**：商品模型，包含商品详情、价格和库存等
3. **Order**：订单模型，关联用户和商品
4. **Cart**：购物车模型，关联用户和商品
5. **Merchant**：商家模型，包含商家信息和店铺详情
6. **Logistics**：物流模型，关联订单和物流信息

### API设计

API采用RESTful风格，以下是主要端点：

#### 用户相关
- `POST /users/register`：用户注册
- `POST /users/login`：用户登录
- `POST /users/profile/get`：获取用户信息
- `POST /users/profile/update`：更新用户信息
- `POST /users/address/manage`：管理收货地址

#### 商品相关
- `POST /merchants/products`：获取商品列表
- `POST /merchants/products/detail`：获取商品详情
- `POST /merchants/products/search`：搜索商品

#### 订单相关
- `POST /users/orders/create`：创建订单
- `POST /users/orders/list`：获取订单列表
- `POST /users/orders/detail`：获取订单详情
- `POST /users/orders/cancel`：取消订单

#### 购物车相关
- `POST /users/cart/add`：添加商品到购物车
- `POST /users/cart/update`：更新购物车商品数量
- `POST /users/cart/remove`：从购物车移除商品
- `POST /users/cart/list`：获取购物车商品列表

## 语音服务后端架构（gzl）

语音服务后端处理语音识别、语音合成和AI对话功能，是连接前端和AI服务的桥梁。

### 技术栈

- **Node.js**：JavaScript运行时
- **Express**：Web应用框架
- **Axios**：HTTP请求处理
- **WebSocket**：实时通信
- **Multer**：文件上传处理

```

│   │   └── speech.controller.js # 语音控制器
│   ├── routes/            # 路由
│   │   ├── coze.routes.js      # Coze路由

│   ├── services/          # 服务
```

### API设计

#### 语音相关
- `POST /speech/recognize`：语音识别
- `POST /speech/coze-to-speech`：AI响应语音合成

#### Coze工作流相关
- `POST /coze/execute`：执行Coze工作流
- `GET /coze/info`：获取Coze工作流信息

## 安全性设计

项目采用以下安全措施：

1. **JWT身份验证**：保护API端点
2. **密码哈希**：使用bcrypt加密存储密码
3. **请求验证**：验证所有API请求参数
4. **CORS配置**：控制跨域资源访问
5. **环境变量**：敏感信息通过环境变量管理
6. **输入净化**：防止SQL注入和XSS攻击

## 可扩展性考虑

系统设计考虑了以下可扩展方面：

1. **模块化设计**：所有功能模块化，便于扩展
2. **微服务架构**：主后端和语音服务分离
3. **可配置API**：API路径和参数可通过配置调整
4. **中间件机制**：使用Express中间件扩展功能
5. **服务抽象**：第三方服务通过抽象层调用，便于替换

## 性能优化

系统设计中考虑了以下性能优化点：

1. **数据缓存**：缓存频繁访问的数据
2. **连接池**：数据库连接池管理
3. **异步处理**：使用Node.js异步特性处理IO密集型任务
4. **请求合并**：减少API请求次数
5. **静态资源优化**：CDN分发和资源压缩

## 监控与日志

推荐集成以下监控和日志方案：

1. **日志收集**：使用Winston或Morgan记录应用日志
2. **性能监控**：集成New Relic或PM2监控应用性能
3. **错误追踪**：集成Sentry等工具追踪和报告错误
4. **API监控**：监控API响应时间和成功率
5. **资源监控**：监控服务器资源使用情况
